In software world we, developers, often share with others. We pack our code into libraries and allow other coders to use them. This increases our productivity as we can focus now on real applications instead of doing core things again and again. This is a standard approach for 2019. It wasn’t like this in the past: there was no Internet and other means of communication wasn’t that effective.

Today, as most of retro devs are using cross development, lack of internet support in your 8-bit machine is no longer an issue. Processing power is also not a problem. So why not incorporate productivity tools such as open source libraries, automated build, continuous integration, automated testing into development process for 8-bit? The c64lib is here to help you.
